你現在看的這個網站，就是這個專案的成品。我不寫前端——整個網站由我與 Claude Code 協作完成，我的貢獻在於定義問題、設下判準、否決不合格的方案。這一頁記錄的是協作過程中「人做了什麼」。

## 問題

這是我職涯轉型路徑上第一個要親自拿給外界看的作品。它要證明的不是我會寫網頁，而是我能利用 AI 解決問題、而且問對問題。設計上我只給了一個方向性要求：視覺要有我的個人特色，而且這個特色要來自「AI 對我的了解」，不是把統計符號貼滿版面的表面做法。

## 我拍板的關鍵決策

**否決 AI 的「預設高級感」。** 第一版視覺是深色底＋暖金色點綴＋方格紙紋理＋散佈圖迴歸線裝飾。好看，但這正是 AI 輔助設計最容易收斂的通用樣板，跟我本人沒有真正的關聯——整版推翻。現在你看到的收斂版本：無紋理、無裝飾性圖形，低調的墨綠 accent 只出現在互動元素上。

**用側寫文件取代規格清單。** AI 曾根據對我的了解提出三個具體互動設計，我否決了直接採用，改為請 AI 先寫一份不含實作建議的人物側寫文件，交由下一輪協作自行詮釋——保留設計的開放性，而不是把 AI 當成執行清單的工具。

**固定深色顯示。** 推翻跟隨訪客系統自動切換的架構。理由誠實說就是個人偏好：我要網站永遠以我選定的樣子呈現。

**內容策略。** 以「是否看得見思考過程」篩選要上架的作品，分類頁在有真實內容前保持留白。這個判準由 AI 提出、我認可採用——不是我原創的框架，如實註明。

## 過程中修正的流程

第一次改版後我在正式網址上看不到變更，查出原因是改動只在本機、尚未部署。此後協作流程固定加上「先確認在看哪個環境」這一步。這次修正後來被制度化：專案內建了決策紀錄協議，任何協作中我做出的拍板、否決、修正，AI 都會即時寫入 repo 的 `DECISIONS.md`，日後每個專案的「人的貢獻」都有原始材料可查，不靠事後回憶。

## 驗證方式（誠實記錄）

目前的驗收是主觀的：親自刷新正式網址，對照「這是不是屬於我自己的樣子」判斷。對比度、跨瀏覽器測試等量化指標尚未建立——這是已知的待補項，不假裝有。

Astro、GitHub Pages（GitHub Actions 自動部署）。完整決策紀錄見 GitHub repo 的 README 與 `DECISIONS.md`。
