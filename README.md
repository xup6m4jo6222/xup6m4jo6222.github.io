# 個人作品集網站

[xup6m4jo6222.github.io](https://xup6m4jo6222.github.io) — AI 專案 #1。

這個網站有雙重身份：它是我統計與 AI 作品的展示頁，同時它本身就是第一個作品——整個網站由我與 AI（Claude Code）協作完成。我不寫前端，我的貢獻在於定義問題、設下判準、否決不合格的方案；具體的程式碼與視覺實作由 AI 產出。這份 README 記錄的就是協作過程中「人做了什麼」。

## 問題定義

這是我職涯轉型路徑上第一個要親自拿給外界看的作品。網站要證明的不是我會寫網頁，而是我能利用 AI 解決問題、而且問對問題。

設計上我只給了一個方向性要求：整體視覺要有我的個人特色（不是指內容），而且這個特色要來自「AI 對我的了解」，不是把統計符號貼滿版面的表面做法。

## 關鍵決策（由我拍板的部分）

- **否決 AI 的「預設高級感」**：第一版視覺是深色底＋暖金色點綴＋方格紙紋理＋散佈圖迴歸線裝飾。好看，但這正是 AI 輔助設計最容易收斂的通用樣板，跟我本人沒有真正的關聯——整版推翻。改版後：拿掉所有紋理與裝飾性圖形，accent 收斂成低調墨綠、只用在互動元素上，邊框改實線 hairline，專案頁改成內文先於標籤的敘事順序。
- **固定深色顯示**：推翻原本跟隨訪客系統設定自動切換淺深色的架構，理由單純是個人偏好，我要網站永遠以我選定的樣子呈現。
- **用側寫文件取代規格清單**：AI 曾根據對我的了解提出三個具體互動設計（捲動畫回歸線、卡片先答後揭等），我否決了直接採用，改為請 AI 先寫一份不含實作建議的人物側寫文件，交由下一輪協作自行詮釋——保留設計的開放性，而不是把 AI 當成執行清單的工具。
- **內容策略**：以「是否看得見思考過程」而非「正式／練習」篩選要上架的作品；分類頁在有真實內容前保持留白，不放施工中文字或假專案卡。這個判準由 AI 提出、我認可採用，不是我原創的框架。

## 過程中修正的流程問題

第一次改版後我在正式網址上看不到變更，查出原因是改動只存在本機、尚未部署。此後協作流程固定加上一步：改動前先確認「現在看的是哪個環境」，再決定何時 commit 與部署。

## 驗證方式

誠實記錄：目前的驗收是主觀的——親自刷新正式網址，對照「這是不是屬於我自己的樣子」判斷，沒有量化指標（對比度、跨瀏覽器測試等尚未建立）。這是已知的待補項，不假裝有。

## 技術

Astro，部署於 GitHub Pages（GitHub Actions 自動建置）。本地開發：`npm install && npm run dev`。
